# Проект: Основы ООП — Интернет-магазин

Проект реализует базовую логику управления товарами, категориями и заказами. В основе архитектуры лежат ключевые принципы объектно-ориентированного программирования (ООП), такие как инкапсуляция, наследование, полиморфизм и абстракция. Также в проекте продемонстрировано применение множественного наследования с использованием классов-миксинов и продвинутая обработка ошибок через пользовательские исключения.

---

## 🛠 Основной функционал и архитектура

### 🔹 1. Абстракция и иерархия сущностей
* **`BaseProduct`** (`src/product.py`) — базовый абстрактный класс для всех продуктов. Он определяет обязательный контракт для дочерних классов, фиксируя требования к инициализации (`__init__`), сложению товаров (`__add__`) и строковому отображению (`__str__`).
* **`BaseCategoryOrder`** (`src/category.py`) — абстрактный класс для группирующих сущностей. Задает единый интерфейс для классов `Category` и `Order` через абстрактные свойства `total_quantity` и `total_cost`.

Классы **«Смартфон»** (`Smartphone`) и **«Трава газонная»** (`LawnGrass`) являются наследниками класса `Product` и автоматически наследуют все свойства абстрактного класса.

### 🔹 2. Множественное наследование и классы-миксины
* **`LogMixin`** — класс-миксин, предназначенный для автоматического логирования. При создании любого объекта (в момент работы метода `__init__`) он автоматически распечатывает в консоль информацию о том, от какого класса и с какими именно параметрами был создан объект.
* **Принцип работы**: Логирование реализовано через переопределение магического метода `__repr__`.
* **MRO**: Миксин успешно добавлен в цепочку множественного наследования класса `Product` с соблюдением порядка разрешения методов (MRO) и корректным использованием `super()`.

### 🔹 3. Управление категориями и заказами
* **Category (Категория)**: Группирует любые типы продуктов. Хранит товары в приватном списке `__products`, полностью защищенном от прямого изменения. Ведет сквозной учет уникальных по названию товаров (`product_count`) и общего числа категорий (`category_count`).
* **Order (Заказ)**: Позволяет оформить покупку одного конкретного товара в заданном количестве. Рассчитывает итоговую стоимость покупки и выводит информативное сообщение в формате: 
  `Заказ: Имя товара x X шт. Итоговая стоимость: Y руб.`

### 🔹 4. Строгий контроль типов и бизнес-логика
* **Валидация добавления (`add_product`)**: Метод на базе `isinstance()` строго контролирует состав категорий и заказов. Попытка передать сторонний объект вызовет ошибку `TypeError`.
* **Безопасное сложение товаров (`__add__`)**: Сложение общей стоимости товаров (цена × количество) доступно **строго для объектов одного и того же класса** (проверка через `type()`). Попытка сложить смартфон и газонную траву вызовет ошибку `TypeError`.

### 🔹 5. Умная полиморфная загрузка данных
* **`read_json`** (`src/utils.py`): Функция динамически считывает JSON-структуру, анализирует состав полей внутри каждого элемента (на предмет наличия специфичных свойств `efficiency` или `germination_period`) и автоматически конструирует объекты целевых классов (`Smartphone`, `LawnGrass` или базовый `Product`).

## 🚨 Обработка исключений и валидация данных

В проекте реализована отказоустойчивая система валидации данных на основе стандартных и кастомных исключений:

### ❌ Контроль нулевого количества товара (`raise ValueError`)
При попытке инициализировать товар с количеством `0` (в методе `__init__`), программа генерирует исключение `ValueError` с сообщением: *«Товар с нулевым количеством не может быть добавлен»* и немедленно прерывает свое выполнение.

### 🧮 Подсчет среднего ценника (`try/except ZeroDivisionError`)
В классе `Category` реализован метод для расчета средней стоимости всех товаров. Метод устойчив к критическим ситуациям: если в категории отсутствуют товары, деление на ноль перехватывается с помощью конструкции `try/except`, предотвращая падение программы и возвращая `0`.

### 🎯 Пользовательские исключения (`try/except/else/finally`)
Для контроля добавления некорректных товаров в «Категорию» или «Заказ» создан индивидуальный класс исключения. Логика обработки спроектирована по полному циклу:
* **`try`**: Происходит попытка добавления товара и проверка его количества.
* **`except`**: В случае нулевого количества вызывается пользовательское исключение, выводящее соответствующее предупреждение в консоль.
* **`else`**: При успешном прохождении всех проверок выводится сообщение о том, что товар успешно добавлен.
* **`finally`**: Независимо от исхода операции, в консоль выводится уведомление о том, что процесс обработки добавления товара завершен.

## 📂 Структура проекта

* `src/product.py` — Абстрактный класс `BaseProduct`, класс-миксин `LogMixin`, базовый класс `Product` и его специализированные наследники (`Smartphone`, `LawnGrass`). Также содержит логику генерации `ValueError` при нулевом количестве.
* `src/category.py` — Абстрактный базовый класс `BaseCategoryOrder`, классы `Category` и `Order`. Включает метод подсчета среднего ценника и кастомные исключения для безопасного добавления товаров.
* `src/utils.py` — Фабричный парсинг JSON-файлов с динамическим определением типов продуктов.
* `tests/` — Полный набор модульных тестов с проверкой абстрактных классов, работы миксина логирования через `capsys`, изоляцией консольного ввода (`mock_input`) и 100% покрытием кода.

## 🚀 Установка и запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com
   cd your-repo-name
   ```

2. Cоздайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Linux/macOS
   # или
   venv\Scripts\activate     # Для Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
