# Проект: Основы ООП — Интернет-магазин

Проект реализует базовую логику управления товарами, категориями и заказами на принципах объектно-ориентированного программирования (ООП), таких как инкапсуляция, наследование, полиморфизм, абстракция, а также множественное наследование с использованием классов-миксинов.

---

## 🛠 Основной функционал и архитектура

### 🔹 1. Абстракция и иерархия сущностей
* **`BaseProduct`** (`src/product.py`) — базовый абстрактный класс для всех продуктов. Он определяет обязательный контракт для дочерних классов, фиксируя требования к инициализации (`__init__`), сложению товаров (`__add__`) и строковому отображению (`__str__`).
* **`BaseCategoryOrder`** (`src/category.py`) — абстрактный класс для группирующих сущностей, задающий интерфейс для классов `Category` и `Order` через абстрактные свойства `total_quantity` и `total_cost`.

Классы **«Смартфон»** (`Smartphone`) и **«Трава газонная»** (`LawnGrass`) остаются наследниками класса `Product` и автоматически наследуют все свойства абстрактного класса.

### 🔹 2. Множественное наследование и классы-миксины (`LogMixin`)
* **`LogMixin`** — класс-миксин, предназначенный для автоматического логирования. При создании любого объекта (в момент работы метода `__init__`) он автоматически распечатывает в консоль информацию о том, от какого класса и с какими именно параметрами был создан объект.
* Логирование реализовано через переопределение магического метода `__repr__`.
* Миксин успешно добавлен в цепочку множественного наследования класса `Product` с соблюдением порядка разрешения методов (MRO) и корректным использованием `super()`.

### 🔹 3. Управление категориями и заказами
* **Category (Категория)**: Группирует любые типы продуктов. Хранит товары в приватном списке `__products`, полностью защищенном от прямого изменения. Ведет сквозной учет уникальных по названию товаров (`product_count`) и общего числа категорий (`category_count`).
* **Order (Заказ)**: Позволяет оформить покупку одного конкретного товара в заданном количестве. Рассчитывает итоговую стоимость покупки и выводит информативное сообщение в формате: `Заказ: Имя товара x X шт. Итоговая стоимость: Y руб.`

### 🔹 4. Строгий контроль типов и бизнес-логика
* **Валидация добавления (`add_product`)**: Метод на базе `isinstance()` строго контролирует состав категорий и заказов. Попытка передать сторонний объект вызовет ошибку `TypeError`.
* **Безопасное сложение товаров (`__add__`)**: Сложение общей стоимости товаров (цена $\times$ количество) доступно **строго для объектов одного и того же класса** (проверка через `type()`). Попытка сложить смартфон и газонную траву вызовет ошибку `TypeError`.

### 🔹 5. Умная полиморфная загрузка данных
* **`read_json`** (`src/utils.py`): Функция динамически считывает JSON-структуру, анализирует состав полей внутри каждого элемента (на предмет наличия специфичных свойств `efficiency` или `germination_period`) и автоматически конструирует объекты целевых классов (`Smartphone`, `LawnGrass` или базовый `Product`).

---

## 📂 Структура проекта

* `src/product.py` — Абстрактный класс `BaseProduct`, класс-миксин `LogMixin`, базовый класс `Product` и его специализированные наследники (`Smartphone`, `LawnGrass`).
* `src/category.py` — Абстрактный базовый класс `BaseCategoryOrder`, управление категориями (`Category`), оформление заказов (`Order`) и ведение сквозной статистики.
* `src/utils.py` — Фабричный парсинг JSON-файлов с динамическим определением типов продуктов.
* `tests/` — Полный набор модульных тестов с проверкой абстрактных классов, работы миксина логирования через `capsys`, изоляцией консольного ввода (`mock_input`) и 100% покрытием кода.

## 🚀 Запуск и тестирование

### Установка окружения
Рекомендуется использовать Python 3.10+ и виртуальное окружение.

```bash
pip install pytest pytest-cov mypy flake8
```

### Запуск проекта (Демонстрация)
Для корректной работы импортов запуск производится из корня проекта:

```bash
python -m src.utils
```

#### Запуск тестов и проверка покрытия
Проект покрыт тестами, проверяющими валидацию цен, корректность сложения стоимости, защиту от смешивания типов и работу фабричных методов.

```bash
# Запуск обычных тестов
pytest

# Запуск тестов с отчетом о покрытии кода (coverage)
pytest --cov=src
```
