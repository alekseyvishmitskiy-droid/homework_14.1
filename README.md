# Проект: Основы ООП — Интернет-магазин

Проект реализует базовую логику управления товарами и категориями на принципах объектно-ориентированного программирования (ООП), таких как инкапсуляция, наследование и полиморфизм. Система поддерживает автоматизированную загрузку данных, строгий контроль типов и модульное тестирование.

---

## 🛠 Основной функционал

### 🔹 Архитектура и наследование классов
Система построена вокруг базового класса и его специализированных наследников:
*   **Product (Товар)**: Базовый класс, инкапсулирующий свойства `name`, `description`, `_price` и `quantity`.
    *   Геттер и сеттер для цены с валидацией (запрет ≤ 0) и подтверждением снижения стоимости через консоль.
    *   Класс-метод `new_product` для «умного» создания объекта или обновления существующего товара в списке (складывание остатков, выбор максимальной цены и бесконфликтное обновление без блокировки тестов).
*   **Smartphone (Смартфон)**: Наследник `Product`. Добавляет характеристики: производительность (`efficiency`, `float`), модель (`model`), объем встроенной памяти (`memory`, `int`) и цвет (`color`).
*   **LawnGrass (Газонная трава)**: Наследник `Product`. Добавляет характеристики: страна-производитель (`country`), срок прорастания (`germination_period`) и цвет (`color`).

### 🔹 Управление категориями и статистика
*   **Category**: Группирует любые типы продуктов. Хранит товары в приватном списке, защищенном от прямого изменения.
*   **Геттер `products`**: Возвращает список красиво форматированных строк с информацией о каждом товаре.
*   **Автоматический учет**: На уровне класса `Category` динамически рассчитывается статистика:
    *   `category_count` — общее количество созданных категорий.
    *   `product_count` — общее количество уникальных товаров во всех категориях.

### 🔹 Строгий контроль типов и бизнес-логика
*   **Безопасное сложение товаров (`__add__`)**: Реализована валидация на основе функции `type()`. Сложение общей стоимости товаров (цена $\times$ количество) доступно **строго для объектов одного и того же класса**. Попытка сложить смартфон и газонную траву вызовет ошибку `TypeError`.
*   **Защита метода `add_product`**: С помощью функции `isinstance()` метод проверяет добавляемый объект. В категорию можно добавлять только экземпляры класса `Product` или его явных наследников (`Smartphone`, `LawnGrass`). Любые сторонние типы вызывают ошибку `TypeError`.

### 🔹 Загрузка данных
*   **read_json**: Функция в `utils.py`, которая читает структуру JSON-файла и на ее основе автоматически конструирует и возвращает готовые объекты классов `Category` и `Product`.

## 📂 Структура проекта
- `src/product.py` — Базовый класс продуктов и специализированные классы-наследники.
- `src/category.py` — Управление категориями, добавление товаров и учет статистики.
- `src/utils.py` — Парсинг JSON-файлов и демонстрационный запуск.
- `tests/` — Полный набор модульных тестов с изоляцией консольного ввода.

## 🚀 Запуск и тестирование

### Установка окружения
Рекомендуется использовать Python 3.10+ и виртуальное окружение.
```bash
pip install pytest pytest-cov mypy flake8
```

### Запуск проекта (Демонстрация)
Для корректной работы импортов запуск производится из корня проекта:
```bash
python -m src.utils
```

### Тестирование и статический анализ
Проект следует строгим стандартам качества кода:
*   **Тесты**: `pytest --cov` (проверка бизнес-логики и покрытия). Все тесты полностью изолированы от интерактивного метода `input()` с помощью автоматических фикстур-заглушек (`unittest.mock.patch`).
*   **Типы**: `mypy src tests` (комплексная проверка статической типизации).
*   **Стиль**: `flake8 src` (проверка соответствия стандарту PEP 8).
